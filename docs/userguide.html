🚀 Getting Started

What is DopaFlow?

DopaFlow is a unified productivity application designed specifically for ADHD brains. It combines task management, habit tracking, focus sessions, spaced repetition review, journaling, calendar planning, and alarms into one cohesive experience with a FastAPI backend, React 18 frontend, and Electron desktop shell.

✓ Tasks

Manage todos with priorities, due dates, subtasks, and dependencies

◎ Habits

Build routines with streak tracking and weekly insights

◴ Focus

Pomodoro-style sessions with momentum tracking

✦ Review

Spaced repetition flashcards for learning

✎ Journal

Daily entries with templates, voice dictation, and spoken command capture

⏰ Alarms

Scheduled reminders with audio and TTS support

Installation

DopaFlow can be used from the installed local release or run from source during development.

Installed local release

./run-dopaflow.sh
Release note: actual launcher names and install paths vary by machine and packaging target. Keep machine-specific wrapper paths in local ops notes, not in the shipped guide.

Run from source

Run from the repository root using the relevant app folder:

Backend (FastAPI)

cd backend
../.venv/bin/python -m app.main

Frontend (React + Vite)

cd frontend
npm install
npm run dev

Desktop (Electron)

cd desktop
npm install
npm run dev

Environment Variables

Configure the backend with these environment variables:

Variable Description Default
DOPAFLOW_DB_PATH SQLite database location backend/dopaflow.sqlite3
DOPAFLOW_DEV_AUTH Bypass auth in development (1 / true / yes) unset (auth enforced)
DOPAFLOW_DISABLE_LOCAL_AUDIO Disable local audio/TTS features false
DOPAFLOW_TURSO_URL Optional Turso cloud sync URL unset (local SQLite only)
DOPAFLOW_TURSO_TOKEN Optional Turso auth token unset
DOPAFLOW_AUTH_TOKEN_SECRET Secret for scoped remote API tokens unset (advanced integrations only)
VITE_API_URL Frontend API base URL Current origin (window.location.origin)

⭐ Core Features

1. Tasks Management

The tasks system is the heart of DopaFlow, featuring:

  • Priority Levels: 1 (highest) to 4 (lowest)
  • Due Dates: Schedule tasks with flexible timing
  • Subtasks: Break down complex tasks
  • Dependencies: Link related tasks
  • Time Tracking: Start/stop timers on tasks
  • Recurring Tasks: Automatic generation from rules
  • Templates: Reusable task patterns
  • Quick Add: Natural language input
  • CSV Import/Export: Bulk operations
Pro Tip: Use Quick Add with natural language like "Call dentist tomorrow at 3pm #health !2" to create tasks with due dates, tags, and priority in one go!

2. Habits Tracking

Build and maintain routines with the habit system:

  • Frequency Goals: Set daily, weekly, or custom targets
  • Streak Tracking: Monitor consecutive completions
  • Freeze Mode: Pause habits without breaking streaks
  • Weekly Insights: Visualize completion patterns
  • Correlations: Discover habit relationships
  • CSV Export: Analyze your data externally

3. Focus Sessions

Pomodoro-style deep work sessions:

  • Custom Durations: Set session length in minutes
  • Task Linking: Associate sessions with specific tasks
  • Session History: Track completed focus blocks
  • Statistics: View productivity trends
  • Recommendations: Get history-based session suggestions from recent focus patterns

4. Spaced Repetition Review

Evidence-based learning with flashcards:

  • Decks: Organize cards by topic
  • SRS Algorithm: SM-2 optimal review scheduling — Again/Hard/Good/Easy ratings update intervals immediately
  • Keyboard Shortcuts: Space or F flips the card; 1/2/3/4 rate the card when flipped — no mouse required for a full review session
  • Session Completion Screen: After clearing the due queue, a summary screen shows how many cards were reviewed and whether more are due in other decks
  • Inline Card Editor: Click Edit on any card (active card face or queue list row) to update the front and back text without leaving the review surface
  • Ease Factors: Adaptive difficulty adjustment
  • Lapse Tracking: Monitor forgotten cards
  • Anki Import: Import .apkg files directly into any deck
  • Deck Export: Share and backup decks

5. Journal

Reflective writing with powerful features:

  • Markdown Support: Rich text formatting
  • Templates: Pre-built entry structures
  • Voice Dictation: Speech-to-text input
  • Wikilinks: Connect related entries
  • Graph View: Visualize entry connections
  • Backlinks: See what links to each entry
  • Backups: Local markdown backup and export controls

6. Obsidian Vault Bridge

Local-first Markdown vault interoperability:

  • Vault Settings: Configure a local Obsidian vault path in Settings
  • Journal Push/Pull: Sync Markdown daily notes in Daily/YYYY-MM-DD.md
  • Task Collections: Sync checkbox task files in Tasks/Inbox.md and per-project task files
  • Task Import: Preview and import task lines from DopaFlow-owned collections and plain markdown task files in the configured task folder
  • Daily Task Section: Push a bounded task section into an existing daily note without overwriting the rest of the note
  • Conflict Tracking: Review indexed conflicts, preview the current vault file against the last DopaFlow snapshot, then rollback or keep the vault version
Compatibility rules: The bridge preserves plain Markdown, Obsidian checkbox syntax, wikilinks, and user-authored content outside DopaFlow-managed sections. Sync is manual-first; there is no live watch or merge UI yet.

7. Calendar Integration

Local planning and shared calendar visibility in one surface:

  • Drag to Reschedule: In the Day view, drag local events vertically to reschedule them — duration is preserved, changes commit on release with 15-minute snapping
  • Resize Events: Drag the handle at the bottom of a local event in Day view to extend or shorten its duration — minimum 15 minutes, commits on release
  • Editable Event Details: Click events from week, day, or month view to inspect and edit local blocks — modal shows source, category, recurrence, and time range at a glance
  • Read-Only Guard: Mirrored events from shared feeds show a lock indicator and cannot be dragged, resized, or edited
  • Quick Block UI: Create local time blocks with category and recurrence directly from the calendar surface — validates date, time, and duration before creating
  • OAuth Flow: Secure Google authentication
  • Mirror Feeds: Shared calendars show source colors and stay read-only in the planning view
  • Conflict Resolution: Sync conflicts can be reviewed and resolved without leaving the calendar
  • Recurring Events: Repeat rules can be created and edited from the UI
  • Category Tags: Organize events by work, personal, health, and focus contexts

8. Alarms & Reminders

Never miss important moments:

  • Flexible Scheduling: One-time or recurring alarms
  • Audio Alerts: Custom alarm sounds
  • TTS Announcements: Text-to-speech reminders
  • YouTube Integration: Play music as alarm
  • Desktop Notifications: System-level alerts

9. Gamification

Motivation through game mechanics:

  • XP System: Earn experience for completions
  • Level Progression: Advance through ranks
  • Shell Progress: See level progress in the top bar during normal use
  • Badges: Unlock achievements for milestones
  • Momentum Score: Track daily productivity flow

10. Packy Memory Support

Contextual memory and guidance system:

  • Ask Mode: Get contextual suggestions from your memory
  • Whisper: Passive motivational nudges
  • Lorebook: Persistent context storage
  • Momentum Tracking: Memory-powered insights

Using the Theme Picker

The theme picker in Settings provides a preview workflow for choosing skins:

  • Single-click any theme card to preview it without applying
  • Double-click to immediately apply a theme
  • Use the Apply button to confirm a previewed theme
  • Use Reset to default to restore Ink & Stone
  • Export saves the current theme as a JSON file for backup or sharing
  • Import loads a custom theme JSON file

The active theme shows "Currently active" status. While previewing, it shows "Previewing (not applied yet)" with a dashed border.

🔒 Security

DopaFlow implements multiple layers of security to protect your data. This section covers authentication, authorization, and data protection measures.

Authentication

DopaFlow uses a flexible authentication system designed for local-first deployment:

  • Local Trust: Requests from localhost (127.0.0.1, localhost) and Electron app protocols (app://, file://) are trusted by default
  • API Key: Remote requests require an x-api-key header when DOPAFLOW_ENFORCE_AUTH is enabled
  • Health Endpoint: The /health endpoint is always accessible for monitoring
Security Warning: The DOPAFLOW_DEV_AUTH environment variable bypasses all authentication when set to 1, true, or yes. Never enable this in production environments.

Authorization Scopes

Fine-grained permission control is available through scope-based authorization:

Scope Description Protected Endpoints
read:tasks Read task data GET /tasks, GET /tasks/{id}
write:tasks Create/modify tasks POST /tasks, PATCH /tasks/{id}, DELETE /tasks/{id}
read:habits Read habit data GET /habits, GET /habits/{id}
write:habits Create/modify habits POST /habits, PATCH /habits/{id}
read:journal Read journal entries GET /journal/entries
write:journal Create/modify journal POST /journal/entries, DELETE /journal/entries/{id}
admin:ops Full ops access /ops/export, /ops/import, /ops/reconcile

Rate Limiting

API rate limiting protects against abuse and excessive resource usage:

  • Default Limit: 60 requests per minute per IP
  • Command Endpoints: Lower limits for AI operations (20-30/min)
  • Ops Endpoints: Strict limits (10/min)
  • Response: HTTP 429 with {"code": "rate_limited", "message": "Too many requests"}

HTTP Security Headers

All API responses include security headers to protect against common web vulnerabilities:

Header Value Purpose
Content-Security-Policy default-src 'none'; script-src 'self'; ... Prevents XSS attacks
X-Frame-Options DENY Prevents clickjacking
X-Content-Type-Options nosniff Prevents MIME-type sniffing
Referrer-Policy strict-origin-when-cross-origin Controls referrer information
Permissions-Policy camera=(), microphone=(), geolocation=() Restricts browser features

Data Protection

Database Security

  • WAL Mode: Write-Ahead Logging prevents database corruption
  • Foreign Keys: Enforced for data integrity
  • Migrations: Version-controlled schema changes

File Upload Security

  • Size Limits: Maximum 10MB per file
  • Type Validation: Suffix and MIME type checking
  • Magic Bytes: Content signature verification for SQLite, ZIP, JSON

CORS Configuration

Cross-Origin Resource Sharing is restricted to trusted origins:

  • app://. - Electron app
  • file:// - Local file access
  • http://localhost:5173-5175 - Development servers
  • http://127.0.0.1:5173-5175 - Development servers
For Production: Keep operational guidance in CHANGELOG.md and LLM_work_folder/ aligned with the live backend configuration before widening network exposure.

🔧 Troubleshooting

Backend won't start

  • Ensure Python 3.11–3.12 is installed (Python 3.13 is not supported)
  • Run pip install -r requirements.txt
  • Check that port 8000 is not in use
  • Verify DOPAFLOW_DB_PATH is writable (defaults to backend/dopaflow.sqlite3)

Frontend can't connect to API

  • Ensure backend is running on port 8000
  • Check VITE_API_URL environment variable
  • Verify CORS settings if using custom origins

Desktop app crashes on startup

  • If running from source, confirm the backend starts successfully first
  • Use the project’s working Node/npm installation rather than relying on temporary local wrappers
  • Review Electron logs only if you are specifically debugging the native shell

Database migration errors

  • Back up your database: cp dopaflow.sqlite3 backup.sqlite3
  • Delete the database to trigger fresh migration
  • Check migration SQL files in backend/migrations/

Audio/TTS not working

  • Ensure DOPAFLOW_DISABLE_LOCAL_AUDIO is not set to true
  • Check system audio permissions
  • Verify browser/site has audio permission
  • For voice commands, check microphone permission and browser speech synthesis support
Need more help? Check the source tree directly, starting with frontend/src/App.tsx, frontend/tests/e2e/, and CHANGELOG.md.
